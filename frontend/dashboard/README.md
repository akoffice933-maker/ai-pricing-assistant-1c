# AI Pricing Assistant — Dashboard

Тонкий веб-клиент поверх backend `ai_pricing_market_mvp`. Вся расчётная логика (кривая
спроса, оптимизация цены) остаётся в Python — дашборд только визуализирует ответы того
же API, которым пользуется 1С.

## Открыть без установки

- https://akoffice933-maker.github.io/ai-pricing-assistant-1c/

Деплоится автоматически (`.github/workflows/pages.yml`) при пуше в `main`. Нужен
запущенный локально backend (см. корневой README, раздел «Веб-версия») — сам по
себе статический хостинг Python не запускает.

## Быстрый запуск (dev)

Backend должен быть запущен отдельно (см. `../../backend/ai_pricing_market_mvp/README.md`).
CORS в backend закрыт по умолчанию — добавьте origin дашборда в `.env` backend'а:

```bash
# backend/ai_pricing_market_mvp/.env
AI_PRICING_ALLOWED_ORIGINS=http://localhost:5173
```

Затем:

```bash
cd frontend/dashboard
npm install
npm run dev
```

Откройте `http://localhost:5173`. В шапке справа — «настройки»: укажите адрес backend'а
(по умолчанию `http://localhost:8000`) и API-токен, если он задан на сервере
(`AI_PRICING_API_TOKEN`). Токен хранится только в localStorage браузера.

## Продакшен-сборка

```bash
npm run build   # -> dist/
```

Либо через Docker (весь стек сразу, из корня репозитория):

```bash
cp .env.example .env   # заполнить реальными значениями; без файла docker compose упадёт
docker compose up --build
```

Дашборд будет на `http://localhost:8080`, backend — на `http://localhost:8000`.

## Что внутри

- **Форма рекомендации** — позиция (товар/услуга), рыночный контекст, бизнес-цель,
  опциональные ограничения (маржа, лимиты изменения цены, округление).
- **Результат** — рекомендованная цена, изменение в %, ожидаемые спрос/выручка/маржа,
  статус (можно применять автоматически / требует согласования).
- **График кривой спроса** — цена → ожидаемый спрос, с закрашенной зоной допустимых цен
  (`price_bounds`) и отмеченной точкой рекомендации. Это визуализация ключевой идеи
  проекта: цена не прогнозируется напрямую, а выбирается по кривой спроса.
- **Объяснение и предупреждения** — те же `explanation`/`warnings`, что видит 1С-менеджер
  в форме согласования.
- **Рыночные индикаторы** — отдельная вкладка: вводите сырые наблюдения конкурентов
  (цена, промо, наличие), получаете нормализованный `market_context` и готовую запись
  для 1С-загрузчика (`one_c_indicator_record`).
- **Сравнение бизнес-целей** — те же позиция и рынок считаются параллельно по всем 6
  целям (прибыль/выручка/доля рынка/распродажа/премиум/загрузка), результат — таблица
  и bar-график рекомендованных цен рядом.

## Стек

React 19 + Vite, Tailwind CSS, Recharts, lucide-react. Без внешнего state-менеджера —
состояние каждой вкладки живёт в её собственном компоненте (`App.jsx` хранит только
активную вкладку).

## Известные ограничения MVP

- Токен хранится в `localStorage` только для localhost-backend (см. `src/api.js`); для
  не-локального backend — только в памяти вкладки, без персистентности.
- `nginx.conf` задаёт CSP и security-заголовки для Docker-деплоя. **GitHub Pages не
  поддерживает кастомные HTTP-заголовки** — версия на Pages идёт без CSP. Если нужен
  публичный доступ с более строгой защитой от XSS, разворачивайте через Docker
  (см. корневой `docker-compose.yml`), а не через Pages.
