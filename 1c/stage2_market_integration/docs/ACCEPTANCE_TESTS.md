# Приёмочные тесты Этапа 2

## Тест 1. FastAPI доступен

1. Запустить `ai_pricing_market_mvp`.
2. В 1С выполнить `AIPricingHttpServer.ПроверитьСоединение()`.
3. Ожидаемо: `status=ok`.

## Тест 2. Загрузка рыночных индикаторов из JSON

1. Открыть форму загрузчика.
2. Вставить `examples/market_indicators_sample.json`.
3. Нажать `Загрузить`.
4. Ожидаемо:
   - создано 2 записи в `AI_РыночныеИндикаторы`;
   - `MarketPriceMedian > 0`;
   - `Confidence > 0`.

## Тест 3. Загрузка рыночных индикаторов из CSV

1. Открыть форму загрузчика.
2. Вставить `examples/market_indicators_sample.csv`.
3. Указать разделитель `;`.
4. Нажать `Загрузить`.
5. Ожидаемо:
   - строки CSV записаны в `AI_РыночныеИндикаторы`;
   - числовые поля корректно преобразованы.

## Тест 4. Расчёт товара

1. Создать/выбрать товар из категории, замаппленной на `wireless_headphones`.
2. Выполнить расчёт.
3. Ожидаемо:
   - в JSON входа есть `item` и `market_context`;
   - в ответе есть `demand_curve`;
   - в регистре заполнены `ExpectedDemand`, `ExpectedRevenue`, `ExpectedGrossProfit`;
   - заполнено `КриваяСпросаJSON`;
   - `RelativePriceCurrent` и `RelativePriceRecommended` рассчитаны.

## Тест 5. Расчёт услуги

1. Выбрать услугу, замаппленную на `1c_implementation_services`.
2. Выполнить расчёт.
3. Ожидаемо:
   - `ItemType = service`;
   - используется market rate median;
   - в JSON входа есть `available_capacity` или `team_utilization_percent`, если они доступны;
   - рекомендация сохраняется.

## Тест 6. Нет рыночных данных

1. Выбрать позицию без маппинга и без индикаторов.
2. Выполнить расчёт.
3. Ожидаемо:
   - сформирован fallback market_context;
   - `confidence` низкий;
   - рекомендация требует ручной проверки.

## Тест 7. Старые рыночные данные

1. Загрузить индикаторы с `data_freshness_days = 60`.
2. Выполнить расчёт.
3. Ожидаемо:
   - confidence ниже;
   - в notes/объяснении есть предупреждение о старых данных.

## Тест 8. Документ изменения цен

1. Выбрать надёжную рекомендацию.
2. Нажать `СоздатьДокументЦен`.
3. Ожидаемо:
   - создан документ `УстановкаЦенНоменклатуры`;
   - документ не проведён;
   - ссылка записана в `AI_РекомендацииПоЦенам.ДокументИзмененияЦен`.

## Тест 9. JSON-аудит

1. Открыть рекомендацию.
2. Проверить поля:
   - `JSONВход`;
   - `JSONВыход`;
   - `MarketContextJSON`;
   - `КриваяСпросаJSON`.
3. Ожидаемо: все поля заполнены и позволяют воспроизвести расчёт.

## Тест 10. Отказ от рекомендации

1. Отклонить рекомендацию.
2. Ввести причину.
3. Ожидаемо:
   - статус `Отклонена`;
   - причина сохранена;
   - JSON-аудит не потерян.
