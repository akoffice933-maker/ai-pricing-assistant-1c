# Спецификация объектов метаданных 1С

## 1. Регистр сведений `AI_РекомендацииПоЦенам`

Тип: **независимый непериодический регистр сведений**.

Назначение: хранение AI-рекомендаций по ценам, статусов пользовательских решений, ссылок на документы и полного JSON-аудита.

### Измерения

| Имя | Тип | Обязательное | Комментарий |
|---|---|---:|---|
| `ИдентификаторРекомендации` | Строка(120) | да | `request_id` из API. Уникальный ключ рекомендации. |
| `Номенклатура` | СправочникСсылка.Номенклатура | да | Товар. |
| `ХарактеристикаНоменклатуры` | СправочникСсылка.ХарактеристикиНоменклатуры / соответствующий тип / можно удалить | нет | Использовать, если конфигурация ведёт характеристики. |
| `ВидЦены` | СправочникСсылка.ВидыЦен / соответствующий тип | нет | Вид цены, для которого рассчитана рекомендация. |

> Если в конфигурации нет характеристик, измерение `ХарактеристикаНоменклатуры` можно убрать, но тогда нужно удалить соответствующие строки из модулей.

### Ресурсы

| Имя | Тип | Комментарий |
|---|---|---|
| `ТекущаяЦена` | Число(15,2) | Цена до рекомендации. |
| `РекомендованнаяЦена` | Число(15,2) | Итоговая цена после бизнес-ограничений. |
| `НижняяГраницаЦены` | Число(15,2) | Нижняя граница доверительного интервала. |
| `ВерхняяГраницаЦены` | Число(15,2) | Верхняя граница доверительного интервала. |
| `ИзменениеПроцент` | Число(10,2) | Изменение цены в %. |
| `ТекущаяМаржа` | Число(10,2) | Текущая маржа, %. |
| `ОжидаемаяМаржа` | Число(10,2) | Прогнозная маржа, %. |
| `ИзменениеМаржи` | Число(10,2) | Изменение маржи, п.п. |
| `ОжидаемыйЭффект` | Число(15,2) | Прогноз изменения прибыли за месяц. |
| `ОжидаемоеИзменениеОбъема` | Число(10,2) | Прогноз изменения объёма, %. |
| `Надежность` | Число(6,4) | Confidence 0..1. |

### Реквизиты

| Имя | Тип | Комментарий |
|---|---|---|
| `ДатаРасчета` | Дата, состав дата+время | Время расчёта. |
| `ДатаИзмененияСтатуса` | Дата, состав дата+время | Последнее изменение статуса. |
| `Статус` | Строка(30) | `Новая`, `Просмотрена`, `Принята`, `Отклонена`, `НаСогласовании`, `Применена`, `Истекла`, `Ошибка`. |
| `УровеньРиска` | Строка(20) | `low`, `medium`, `high`. |
| `Стратегия` | Строка(50) | Например `maximize_margin`. |
| `РекомендованноеДействие` | Строка(80) | Например `create_price_change_draft`, `manual_review`. |
| `Приоритет` | Строка(20) | `normal`, `high`. |
| `СрокДействияДней` | Число(3,0) | Обычно 7 или 14. |
| `ТребуетСогласования` | Булево | Из ответа API. |
| `ВерсияМодели` | Строка(80) | Например `pricing_v1.3.0`. |
| `КраткоеОбъяснение` | Строка неограниченной длины | Summary. |
| `ПоложительныеФакторы` | Строка неограниченной длины | Список факторов через перевод строки. |
| `НегативныеФакторы` | Строка неограниченной длины | Список факторов через перевод строки. |
| `НейтральныеФакторы` | Строка неограниченной длины | Список факторов через перевод строки. |
| `Предупреждения` | Строка неограниченной длины | Warnings. |
| `JSONВход` | Строка неограниченной длины | Полный JSON-запрос в FastAPI. |
| `JSONВыход` | Строка неограниченной длины | Полный JSON-ответ FastAPI. |
| `Пользователь` | Строка(150) | Пользователь, инициировавший расчёт или действие. |
| `ПричинаОтклонения` | Строка неограниченной длины | Обязательна при отклонении. |
| `ДокументИзмененияЦен` | ДокументСсылка.УстановкаЦенНоменклатуры | Созданный черновик документа. |
| `ДатаПринятия` | Дата, состав дата+время | Когда пользователь принял рекомендацию. |
| `ДатаПрименения` | Дата, состав дата+время | Когда цена фактически применена. |
| `ФактическийЭффект` | Число(15,2) | Заполняется после контрольного периода. |
| `ПоследняяОшибка` | Строка неограниченной длины | Техническая ошибка расчёта/интеграции. |

### Индексы

Рекомендуемые индексы:

1. `ИдентификаторРекомендации`.
2. `Номенклатура`, `ДатаРасчета`.
3. `Статус`, `ДатаРасчета`.
4. `ВидЦены`, `Статус`.

## 2. Константы настроек

Минимальный набор:

| Имя | Тип | Значение по умолчанию |
|---|---|---|
| `AI_Pricing_Host` | Строка(255) | `localhost` |
| `AI_Pricing_Port` | Число(5,0) | `8000` |
| `AI_Pricing_UseHTTPS` | Булево | `Ложь` |
| `AI_Pricing_BasePath` | Строка(100) | пусто |
| `AI_Pricing_APIToken` | Строка неограниченной длины | пусто |
| `AI_Pricing_Timeout` | Число(5,0) | `30` |
| `AI_Pricing_MinConfidence` | Число(4,2) | `0.70` |
| `AI_Pricing_MaxIncreasePercent` | Число(5,2) | `20` |
| `AI_Pricing_MaxDecreasePercent` | Число(5,2) | `30` |
| `AI_Pricing_MinMarginPercent` | Число(5,2) | `10` |
| `AI_Pricing_PriceEnding` | Число(3,2) | `0.99` |

> В production API-токен лучше хранить не в открытой константе, а в защищённом хранилище/на стороне сервера 1С с ограниченными правами.

## 4. Обработка/форма

Создать внешнюю обработку или объект расширения:

`AI_РекомендацииПоЦенам`

Формы:

1. `ФормаСписка` — список рекомендаций, фильтры, команды.
2. опционально `ФормаКарточки` — детальный просмотр рекомендации.

Описание формы — в `FORM_SPEC.md`.
