# Production notes для внедрения AI Pricing Assistant в 1С

## 1. Где потребуется адаптация

В первую очередь нужно адаптировать `AIPricingDataProviderServer.bsl`:

- `ПолучитьТекущуюЦену`;
- `ПолучитьСебестоимость`;
- `ПолучитьОстаток`;
- `ПолучитьПродажиЗаПериод`;
- `ПолучитьДанныеКонкурентов`;
- `ТоварУчаствуетВАкции`;
- `ЦенаЗаблокирована`.

Эти источники различаются в УТ 11, ERP, КА, Рознице и доработанных базах.

## 2. Регистр сведений как аудит

`AI_РекомендацииПоЦенам` — это не просто таблица результата. Это журнал принятия решений:

- какие входные данные были переданы;
- какой ответ вернула модель;
- какая версия модели использовалась;
- кто принял/отклонил рекомендацию;
- какой документ создан;
- что произошло после применения.

## 3. Не проводить документ автоматически

Даже если рекомендация надёжная, MVP должен только создать черновик документа.

Правильная цепочка:

```text
AI рассчитал → пользователь проверил → создан черновик → человек провёл/согласовал
```

## 4. Роли и права

Рекомендуемые роли:

- `AI_ПросмотрРекомендаций`;
- `AI_РасчетРекомендаций`;
- `AI_УправлениеРекомендациями`;
- `AI_СогласованиеЦен`;
- `AI_Администратор`.

## 5. Контроль качества

Через 14/30 дней после применения цены нужно сравнивать прогноз и факт:

- продажи;
- выручка;
- валовая прибыль;
- маржа;
- остаток;
- отклонение от ожидаемого эффекта.

Это можно добавить отдельным регламентным заданием.

## 6. LLM-слой

LLM можно подключать позже. В production-роли она должна:

- понимать вопрос пользователя;
- выбирать навык;
- объяснять ответ;
- формировать текстовые summary.

LLM не должна:

- сама считать цену;
- менять регистры;
- проводить документы;
- обходить бизнес-ограничения.
