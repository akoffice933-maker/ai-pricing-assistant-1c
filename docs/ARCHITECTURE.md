# Архитектура AI Pricing Assistant для 1С

## Основной принцип

```text
1С хранит данные и документы.
FastAPI Skills Layer выполняет расчёты.
LLM управляет диалогом и вызывает skills.
Пользователь принимает решение.
```

## Pricing pipeline

```text
1С internal data
цены, продажи, себестоимость, остатки/мощность
        │
        ▼
Market Context
медиана рынка, общий спрос, промо, доступность, сезонность
        │
        ▼
Demand Curve Skill
Q(price / market_price)
        │
        ▼
Price Optimizer
profit/revenue/share/utilization goal
        │
        ▼
1С recommendation audit
JSON вход/выход, кривая спроса, статус, документ
```

## Почему не predict_price

Цена сама по себе не является независимой целевой переменной. Бизнесу нужна реакция спроса на цену относительно рынка.

Поэтому правильнее:

```text
forecast_demand_curve(context, price_grid) → Q(p)
optimize_price(Q(p), constraints, goal) → price
```

## 1С-контур

- `AI_РекомендацииПоЦенам` — журнал рекомендаций и решений.
- `AI_РыночныеИндикаторы` — market context.
- `AI_МаппингРыночныхКатегорий` — связь номенклатуры 1С с рыночной категорией.
- `УстановкаЦенНоменклатуры` — создаётся непроведённым документом.

## Security principle

AI не проводит документы автоматически. Любое изменение цены подтверждается человеком или стандартным согласованием 1С.
